## 注意

2026/02/27：**因项目使用率不高，我决定暂停维护，以下限制请务必注意**：

### ​导入 ZIP 的安全性

导入功能会将 ZIP 文件解压并读取其中的 configs.json，但并未对文件内容做充分校验。**请勿导入任何非自己创建的 ZIP 文件**，恶意构造的文件可能在您的系统中任意增删文件（权限允许范围内）。

### ​默认路径的跨机兼容性

当您在某台电脑上将 Krita 资源路径设置为 Krita 的默认路径（如 %APPDATA%\krita）并导出配置为 .zip 后，如果将此 ZIP 导入到另一台电脑，程序仅处理 {$USERDIR} 占位符。

这意味着，如果新电脑的默认路径因系统语言、账户类型（本地/微软账户）、系统盘符不同而与原电脑不一致，则导入后的配置可能指向错误位置（也可能不错误，这个Bug太诡异了，我不知道该如何修改），导致 Krita 无法找到资源。此时您需要手动将文件移动到正确的路径。

一个常见的例子：原电脑使用在线账户，新电脑先使用本地账户后登录微软账户，导致用户目录分散在两处，程序无法自动修正。

建议：在跨机传输配置时，确保目标电脑的 Krita 资源路径与源电脑一致，或者导入后检查并手动调整路径。

## 语言支持

- ​**多语言本地化**​：所有 GUI 显示文本均从 `./language/*.json` 动态加载。`ZH.json` 作为默认备用文件，若缺失将自动创建。当前已包含 `EN.json`。如您的语言未支持，可编辑/创建对应文件并提交。

## 使用指南

功能直观简单：

1. 点击左下角 `+`/`-` 按钮创建/管理配置组
2. 点击 "应用" 切换 Krita 配置
3. 使用右上角 `设置` 按钮配置工具及查看信息
4. 使用 `导入/导出` 按钮共享配置

## 开发说明

安装依赖：

```
pip install sv_ttk pywinstyles pillow darkdetect
```

​**打包说明**​（使用 `pyinstaller`/`nuitka` 等）：  
打包后需将 `./resources` 和 `./language` 文件夹复制到根目录（与 .exe 文件同级）。

Windows 带图标打包命令（使用 `pyinstaller`）：

```
pyinstaller main.spec
```
